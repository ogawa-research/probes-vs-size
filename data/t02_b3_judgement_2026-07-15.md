## 1. 判定規則（事前固定の確認）

1. **閾値**: Qwen3（instruct checkpoint）= **85.6**・Gemma-3（`-pt` base checkpoint・B2 因子測定と同一）= **79.4**（RULER 論文 Appendix F の base 用）。混在は指示書 §7 裁定 1 のとおり開示 + 感度分析。
2. **effective length** = 4K から**連続して** 13 タスク平均が閾値以上である最長の測定長（連続超過規則）。最短 4K で不通過なら「< 4096」（左打ち切り・censored）。
   - 連続規則を明文化する理由: 感度分析（Gemma-3 を 85.6 で切る場合）で gemma-3-12b が非単調通過（32K のみ通過・4K–16K 不通過）になるケースが実在する。単純な「通過する最長点」規則では孤立通過点を effective と誤認する。主判定（事前固定閾値）の範囲では両規則の結果は一致し、この明文化は判定を動かさない。
3. **条件付き延長**: 32K 超過 → 64K、64K 超過 → 128K（27b のみ skip 条項あり・未発動）。128K 不通過につき 256K なし——両モデルとも規則どおり停止。
4. **プロトコル**: answer_prefix 復元パッチ（論文時点プロトコル・2026-07-13 裁定）を全 7 モデルに統一適用。`protocol: "answer_prefix_restored"` を全 JSON で確認。

## 4. 開示事項

1. **27b 128K の cwe 急落**: 66.62（64K）→ 2.7（128K）。13 タスク平均への機械適用なので判定は不変だが、128K 不通過の主因が cwe 一点に偏る（cwe を除く 12 タスク平均は 78.75 で、それでも 79.4 未満——cwe 単独では説明しきれない全体的低下も併存）。
2. **12b の vt 異常**: 4K = 0.4 → 32K = 53.56 → 128K = 12.28 の非単調。短長で vt がほぼゼロなのは 12b 固有（27b は 4K で 89.72）。タスク内訳の不均一性として記録——平均への機械適用は維持。
3. **平均の非単調**: 12b（32K 86.44 > 16K 83.41）・1b（16K 41.17 > 8K 38.38）。連続超過規則の範囲では判定に影響なし。
4. **noprefix 副産物**: qwen3-0.6b・gemma-3-270m はパッチ前（answer_prefix なし）の完走データを `*_noprefix*` として退避済み。プロトコル感度の実測: 4K 平均で qwen3-0.6b 75.29 → 80.44・gemma-3-270m 6.48 → 37.51（niah 系がほぼ全滅から回復）——answer_prefix 除去が 2025-01-22 リファクタ由来の回帰であり base モデルに致命的、という 2026-07-13 裁定の定量的裏づけ。noprefix 値は判定に不使用。
5. **NVIDIA 公開値とのプロトコル同一性は外部から確定不能**（2026-07-13 裁定のまま）。本測定の 7 点は answer_prefix 復元プロトコルで内的に統一されており、B4 の回帰は内的比較可能性の上に立つ。公開値行（他モデル）と混在させる場合はプロトコル差を開示する。
6. **検証の範囲**: 研究者は per-task スコア → 13 タスク平均 → 閾値適用を独立再計算した（34/34 一致）。per-task スコア自体（公式 evaluator の pred → スコア変換）は再実行していない——pred 生データ（tar・sha256 記録済み）から再評価可能な状態で保全されている。
7. **測定条件の同一性**: 64K/128K 追加測定を含め全長で vLLM 設定は既定値のまま（12b/27b とも KV 容量エラー・設定調整なし・RUN_LOG 記載）——長さ間の測定条件差はない。
